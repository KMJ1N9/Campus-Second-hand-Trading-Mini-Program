# 校园二手交易小程序 — 项目设计方案

> **课程**：移动UI应用开发 | **题目**：二手交易 | **平台**：微信小程序
> **设计系统来源**：ui-ux-pro-max 技能（Marketplace P2P + Vibrant & Block-based + Flat Design）

---

## 一、需求分析（15分）

### 1.1 项目背景

大学生群体每年产生大量闲置物品（教材、电子产品、生活用品等），传统的校园二手交易依赖QQ群、微信群、贴吧等方式，存在信息杂乱、信任缺失、交易效率低等问题。本项目旨在开发一款**专为校园场景设计的二手交易微信小程序**，提供规范化的商品发布、浏览、沟通和交易管理功能。

### 1.2 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **买家** | 在校大学生，寻找高性价比二手商品 | 快速找到想要的商品、了解商品真实情况、安全交易 |
| **卖家** | 在校大学生，出售闲置物品 | 便捷发布商品、管理在售商品、高效沟通 |
| **管理员** | 平台维护者（可选） | 审核违规商品、管理用户、数据统计 |

### 1.3 功能需求全景

```
校园二手交易小程序
├── 1. 用户系统
│   ├── 1.1 微信一键登录（wx.login + 授权）
│   ├── 1.2 用户资料编辑（头像、昵称、联系方式）
│   └── 1.3 身份认证（学号/校园卡，提升信任度）
│
├── 2. 商品管理（卖家端）
│   ├── 2.1 商品发布（图片上传、标题、描述、价格、分类、成色）
│   ├── 2.2 商品编辑/下架/删除
│   ├── 2.3 商品状态管理（在售/已预订/已售出）
│   └── 2.4 我的发布列表
│
├── 3. 商品浏览与搜索（买家端）
│   ├── 3.1 首页商品瀑布流/卡片列表
│   ├── 3.2 分类浏览（教材、电子、生活、服饰、运动、其他）
│   ├── 3.3 关键词搜索 + 筛选排序（价格、发布时间）
│   ├── 3.4 商品详情页（图片轮播、描述、卖家信息）
│   └── 3.5 商品收藏
│
├── 4. 交易沟通
│   ├── 4.1 联系卖家（在线聊天/获取联系方式）
│   ├── 4.2 留言评论
│   └── 4.3 举报不良信息
│
├── 5. 个人中心
│   ├── 5.1 我的收藏
│   ├── 5.2 我的发布（在售/已售出）
│   ├── 5.3 浏览历史
│   ├── 5.4 消息通知
│   └── 5.5 设置
│
└── 6. 系统基础
    ├── 6.1 数据缓存与离线展示
    ├── 6.2 图片懒加载
    ├── 6.3 空状态/错误状态提示
    └── 6.4 下拉刷新 / 上拉加载更多
```

### 1.4 核心用例流程图

**买家购买流程**：
```
打开小程序 → 浏览首页/搜索 → 点击商品 → 查看详情
    → 点击"联系卖家" → 在线沟通 → 线下交易 → 完成
```

**卖家发布流程**：
```
打开小程序 → 个人中心 → 发布商品 → 上传图片
    → 填写信息（标题/价格/分类/成色/描述）→ 提交发布 → 商品上架
```

## 三、数据库设计

### 3.1 数据库选型

采用微信云开发提供的**云数据库（文档型 NoSQL）**，类 MongoDB，支持 JSON 文档存储，每个集合自动生成 `_id`（ObjectId）。

### 3.2 集合设计（4张核心表）

#### 3.2.1 用户集合 `users`

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `_id` | string | ✓ | 用户 openid（由微信登录自动获取） |
| `nickname` | string | ✓ | 昵称 |
| `avatar` | string | ✓ | 头像URL |
| `phone` | string | | 手机号 |
| `wechat_id` | string | | 微信号 |
| `school` | string | | 学校 |
| `student_id` | string | | 学号（认证用） |
| `is_auth` | boolean | | 是否已认证（默认false） |
| `create_time` | date | ✓ | 注册时间 |
| `update_time` | date | ✓ | 更新时间 |

**索引**：`_id`(主键), `{student_id: 1}`(唯一,稀疏)

#### 3.2.2 商品集合 `goods`

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `_id` | string | ✓ | 自动生成 |
| `title` | string | ✓ | 商品标题（≤30字） |
| `description` | string | ✓ | 商品描述（≤500字） |
| `price` | number | ✓ | 价格（元，保留2位小数） |
| `original_price` | number | | 原价 |
| `images` | array[string] | ✓ | 图片URL列表（1-9张） |
| `category` | string | ✓ | 分类：`textbook`/`digital`/`life`/`clothes`/`sport`/`other` |
| `condition` | string | ✓ | 成色：`new`/`like_new`/`used`/`old` |
| `status` | string | ✓ | 状态：`selling`/`reserved`/`sold` |
| `seller_id` | string | ✓ | 卖家openid |
| `view_count` | number | | 浏览量（默认0） |
| `like_count` | number | | 收藏数（默认0） |
| `location` | string | | 交易地点（如"xx校区xx宿舍"） |
| `tags` | array[string] | | 标签（如["考研","英语"]） |
| `create_time` | date | ✓ | 发布时间 |
| `update_time` | date | ✓ | 更新时间 |

**索引**：`_id`(主键), `{seller_id: 1, create_time: -1}`, `{category: 1, create_time: -1}`, `{status: 1, create_time: -1}`, `{title: 'text', description: 'text'}`(文本搜索)

#### 3.2.3 收藏集合 `collections`

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `_id` | string | ✓ | 自动生成 |
| `user_id` | string | ✓ | 用户openid |
| `goods_id` | string | ✓ | 商品ID |
| `create_time` | date | ✓ | 收藏时间 |

**索引**：`{user_id: 1, goods_id: 1}`(唯一复合索引)，`{user_id: 1, create_time: -1}`

#### 3.2.4 聊天消息集合 `messages`

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `_id` | string | ✓ | 自动生成 |
| `from_id` | string | ✓ | 发送者openid |
| `to_id` | string | ✓ | 接收者openid |
| `goods_id` | string | | 关联商品ID |
| `content` | string | ✓ | 消息内容 |
| `type` | string | ✓ | 类型：`text`/`image` |
| `is_read` | boolean | | 是否已读（默认false） |
| `create_time` | date | ✓ | 发送时间 |

**索引**：`{from_id: 1, to_id: 1, create_time: -1}`, `{to_id: 1, is_read: 1}`

### 3.3 ER实体关系简要说明

```
用户(users) 1 ──── N 商品(goods)     [一个用户发布多个商品]
用户(users) 1 ──── N 收藏(collections) [一个用户收藏多个商品]
商品(goods) 1 ──── N 收藏(collections) [一个商品被多个用户收藏]
用户(users) 1 ──── N 消息(messages)    [用户发送/接收多条消息]
```

### 3.4 数据安全规则

- `users`：用户仅可**读**自己数据，可**写**自己数据
- `goods`：所有用户可**读**，仅创建者可**写/删**
- `collections`：用户仅可**读/写**自己的收藏
- `messages`：用户仅可**读**自己的消息，可**写**自己发送的消息

## 四、UI/UX 设计

> 以下设计系统由 **ui-ux-pro-max 技能** 自动生成并结合微信小程序规范调整而来。

### 4.1 设计策略（产品匹配）

根据 `skills-main` 前端设计指导 + `ui-ux-pro-max` 搜索引擎匹配结果：

- **产品类型**：Marketplace (P2P) — 二手交易平台
- **主风格**：Flat Design + Bento Grids 融合（现代简洁，卡片化布局）
- **辅风格**：微交互（Micro-interactions）提升操作反馈
- **目标用户**：18-25岁在校大学生，偏好活泼但不失信任感的界面

### 4.2 色彩系统

采用 Marketplace (P2P) 专业配色方案（紫色系 = 信任 + 绿色 = 成交）：

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-primary` | `#7C3AED` | 主色（按钮、标签、强调） |
| `--color-on-primary` | `#FFFFFF` | 主色上的文字 |
| `--color-secondary` | `#A78BFA` | 辅色（次要元素） |
| `--color-accent` | `#16A34A` | 强调色（价格、成交标签） |
| `--color-background` | `#FAF5FF` | 页面背景 |
| `--color-surface` | `#FFFFFF` | 卡片/表面背景 |
| `--color-foreground` | `#1E293B` | 主文字（深灰，非纯黑） |
| `--color-muted` | `#ECEEF9` | 弱化背景 |
| `--color-muted-foreground` | `#64748B` | 辅助/弱化文字 |
| `--color-border` | `#E2E8F0` | 边框/分割线 |
| `--color-destructive` | `#DC2626` | 删除/警告 |
| `--color-success` | `#16A34A` | 成功/已完成 |

**WCAG 对比度验证**：主文字 #1E293B 对 #FAF5FF 背景 = 13.1:1 ✓（远高于4.5:1标准）

### 4.3 字体系统

微信小程序使用**系统字体栈**（Google Fonts 不适用于小程序），按优先级降级：

```css
/* 全局字体Token */
--font-heading: -apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", sans-serif;
--font-body: -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif;
--font-price: "SF Mono", "Menlo", monospace;  /* 价格使用等宽字体防抖动 */
```

**字号阶梯**（基于 4pt 递增，适配 375px iPhone 基准）：

| 级别 | 字号 | 用途 |
|------|:---:|------|
| `text-xs` | 22rpx | 标签、角标、辅助信息 |
| `text-sm` | 26rpx | 描述文字、时间 |
| `text-base` | 28rpx | 正文 |
| `text-lg` | 32rpx | 标题、导航 |
| `text-xl` | 36rpx | 页面大标题 |
| `text-2xl` | 40rpx | 价格数字 |
| `text-3xl` | 48rpx | 首页推荐位标题 |

### 4.4 间距系统

遵循 **8pt 原子间距系统**（`ux` 域的 `spacing-scale` 规则）：

```
--space-1:  8rpx    (最小间距)
--space-2:  16rpx   (元素内间距)
--space-3:  24rpx   (组间距)
--space-4:  32rpx   (块间距)
--space-5:  48rpx   (段落间距)
--space-6:  64rpx   (大区块间距)
```

**微信小程序适配**：1pt ≈ 2rpx（在 375px 屏宽下），使用 rpx 单位保证不同屏幕等比缩放。

### 4.5 圆角与阴影

遵循 Flat Design + Bento Grids 融合风格：

| 元素 | 圆角 | 说明 |
|------|:---:|------|
| 商品卡片 | 16rpx | Bento Grids 风格 |
| 按钮 | 12rpx | 大按钮使用全圆角 50rpx |
| 图片 | 12rpx | 略小于卡片 |
| 搜索栏 | 50rpx | 胶囊型 |
| 标签/角标 | 8rpx | 小元素 |

**阴影**（仅卡片使用，Flat Design 限制）：`box-shadow: 0 2rpx 12rpx rgba(0,0,0,0.06)`

### 4.6 图标方案

遵循 `ui-ux-pro-max` 技能 **No Emoji as Structural Icons** 规则，使用：
- 微信小程序内置 `<icon>` 组件
- 自定义 SVG 图标（统一 24×24pt 画布，1.5px 描边）

### 4.8 交互规范

遵循 `ui-ux-pro-max` **Interaction (App)** 规范表：

| 交互 | 规范 | 本项目实现 |
|------|------|----------|
| **按下反馈** | 80-150ms 内提供视觉反馈 | 卡片按下 opacity 0.7；按钮颜色加深 scale 0.97 |
| **动画时长** | 微交互 150-300ms | 页面过渡 250ms ease-out |
| **触摸目标** | ≥44×44pt（88×88rpx） | 所有可点击区域 ≥ 88rpx |
| **触摸间距** | 相邻元素 ≥16rpx | 列表项间距 ≥20rpx |
| **禁用状态** | opacity 0.38-0.5 + 无点击响应 | 未填完表单时按钮半透明 |
| **加载反馈** | 骨架屏 + 加载中状态 | 列表骨架屏 + 提交按钮loading |
| **空状态** | 帮助文案 + 引导行动 | "还没有商品，去发布吧～" + 按钮 |
| **错误恢复** | 错误信息 + 重试路径 | Toast提示 + 网络重试按钮 |
| **确认操作** | 删除前二次确认 | 弹窗确认"确定删除该商品？" |
| **下拉刷新** | 原生动画 | 首页 + 列表页启用 |
| **安全区** | 避开刘海+底部指示条 | 使用 `safe-area-inset-bottom` |
| **减动模式** | 尊重系统设置 | 检测 `prefers-reduced-motion` |
